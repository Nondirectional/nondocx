# 协商与专家实施分离

## 目标

将 Demo 从“每条聊天消息立即 ANALYZE → PLAN → COMMIT”改为两阶段工作流：主 Agent 先与用户读取文档、澄清和协商；仅在主 Agent 明确请求、用户点击“开始实施”授权后，才临时规划并调度按工具组拆分的 LLM 专家实施。

## 已确认事实

- 当前 Demo 将用户消息直接交给单一 `LlmDocxExpert`，`RouterAgent` 在同次调用中构建快照、规划、review 并直接提交。
- 当前专家只产生 `Operation`，实际写入由 `CommitCoordinator` 与各工具组 `OperationExecutor` 集中执行。
- Demo 已通过 SSE 显示专家 Prompt、流式 response/thinking 与阶段事件；`DocxToolkit` 为单 Agent、非线程安全实例。
- 可执行工具组为 `body`、`table`、`header-toc`、`revision`、`quality`；`session`、`view`、`capability` 不产生写入操作。

## 需求

1. 新增持久对话上下文的主 Agent，负责协商、解释、只读检索与提出实施请求；不持有写权限。
2. 主 Agent 可使用跨工具组的 `READ/QUERY` 工具，优先语义 `view_*`；不得调用 `CREATE/UPDATE/DELETE`。
3. 协商不创建或版本化用户可见草案；主 Agent 仅在信息足够时提出实施请求。
4. 前端以“开始实施”按钮作为唯一授权入口；普通文本不触发执行。授权记录必须写入 trace。
5. 授权后主 Agent 临时生成 `DispatchPlan`：写明目标工具组、各组任务、验收条件与依赖；禁止关键词 `relevantTo()` 自行猜测分流。
6. 以 `body`、`table`、`header-toc`、`revision`、`quality` 各一个 LLM 专家替换单一通用 LLM 专家。专家只输出本组 `Operation`，不直接写文档。
7. 无依赖专家可并行规划；有依赖专家遵循 `DispatchPlan` 顺序。所有写操作仍由集中提交器串行执行。
8. 授权后重新获取最新快照；提交前校验文档代次/指纹。文档上传、重开或外部变更时取消本次实施并回到协商。
9. 一次实施为事务：任何规划、review 或操作失败，以及用户取消，均回滚本次全部改动且不保存文档。
10. 专家发现歧义、冲突或能力缺口时停止并返回结构化阻塞原因；主 Agent 重新协商，须再次授权。
11. 成功提交后强制重新快照并由 `quality` 专家按 `DispatchPlan` 验收；质量失败只报告，不自动修复。
12. Demo 默认完整展示主 Agent、专家、工具调用、`DispatchPlan`、review、提交和质量验收的全过程。SSE 之外按会话写入本地 JSONL，页面重载后可回放；严禁记录 API Key。
13. 主 Agent 与专家默认共享一个可配置 LLM provider/model，但角色 prompt、上下文和工具白名单独立，且允许角色级覆盖。
14. 专家每次实施无跨轮记忆，只接收分配任务、验收条件、全局约束和相关文档片段，不接收完整协商聊天记录。
15. 实施中禁用再次授权，提供“取消实施”；在操作边界检测取消并回滚。普通新消息排队，实施结束后回到协商。
16. 彻底移除旧的自动执行高层入口与旧 `RouterAgent.run()` 流程；保留并改造 `Operation`、`MergedPlan`、`CommitCoordinator` 等已验证内部协议。

## 验收标准

- [ ] 用户可多轮读取、讨论文档，未点击“开始实施”时零写入、零专家实施调度。
- [ ] 按钮授权后，trace 可见主 Agent 的 `DispatchPlan`、每个被派发专家及其完整流式过程。
- [ ] 跨工具组需求仅唤起 `DispatchPlan` 指定专家；独立专家并行规划，写操作保持集中串行。
- [ ] 规划/冲突/执行失败、用户取消、文档变更均不会留下部分修改或保存文件，并返回可继续协商的结构化原因。
- [ ] 成功提交后执行质量验收；验收未过不自动修复。
- [ ] 任意会话的完整 trace 可持久化为 JSONL 并在重载后回放，且不含密钥。
- [ ] 删除旧“一条消息自动实施”路径，Demo 只走新协商编排入口。
- [ ] 自动化端到端测试至少覆盖：仅读取协商、未授权零写入、授权分派、跨组并发规划、冲突阻断、失败回滚、取消回滚、提交后质量验收、trace 回放。

## 不在范围

- 让专家直接调用写工具或绕过集中提交器。
- 未经用户再次授权的自动修复。
- 为每个专家预先配置不同模型；仅保留角色级配置扩展点。
