# inspect-special-parts

## 触发场景

用户想了解页眉、页脚、目录等特殊部分的现状，而非修改正文。例如：
"页眉写了什么"、"有没有目录"、"页脚是什么内容"。

## 操作顺序

1. 先 `current_document` 取 doc_id。
2. 用只读检查工具读取页眉/页脚/目录内容与结构。
3. 如实汇报读到的内容；读不到就说该部分为空或不存在。

## 边界

- 严格只读：本 Demo 暂不支持页眉、页脚、目录的写入工具。
- 对写入请求必须如实说明"Demo 暂不支持写入此处"，绝不伪造修改成功。
- 目录刷新依赖 Word 分页引擎，本 Demo 不提供刷新能力，需如实告知。
- 不同节（section）可能有不同页眉页脚，读取时按节说明来源，不混为一谈。

## 停止 / 报告条件

- 汇报完特殊部分事实后结束；若用户要改这些部分，说明不支持并建议范围外的处理方式。
- 不把"读到了"误报成"可以改"；读取与写入能力边界必须对用户清晰。
- 当目录条目与实际标题不一致时，以工具读到的原文为准并提示可能的刷新需求。
