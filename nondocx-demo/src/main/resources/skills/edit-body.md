# edit-body

## 触发场景

用户要求修改正文：添加/替换/删除段落、调整 run 样式、加超链接。例如：
"在开头加个居中标题"、"把第二段改成加粗"、"删掉空段落"、"给这段加书签链接"。

## 操作顺序

1. 先 `current_document` 取 doc_id，必要时 `view_body` 确认目标段落索引。
2. 按改动类型调对应写工具：`insert_paragraph`/`replace_paragraph`/`delete_paragraph`/`update_run_style`/`add_or_replace_hyperlink`。
3. 改动后用 `check_quality` 复核，确认无 error。
4. 循环结束由应用层强制保存，你不需要也不能调用保存工具。

## 边界

- 只处理正文段落、run、样式、超链接；表格内部改动归 `edit-table`，修订语义归 `tracked-changes`。
- 写工具调用会把本轮标记 dirty，触发应用层落盘门控（质检 error 拒绝、warning 允许）。
- 不回显被改全文，写工具结果已瘦身为一行确认，无需重复粘贴段落原文。
- 先定位再改：改动前用只读工具确认目标索引，避免改错段落。

## 停止 / 报告条件

- 改动完成并通过质检后结束；质检有 error 时应用层会回滚，应据实报告未保存。
- 若用户请求的能力超出正文范围（如页眉页脚写入），如实说明不支持，不伪造成功。
- 单轮内同类写操作应聚合成最少调用，避免反复插入/删除同一段造成抖动。
