# inspect-document

## 触发场景

用户想了解文档现状、定位某段内容、确认结构，而非直接改动。例如：
"这文档讲了什么"、"第几段是标题"、"哪里提到了 XXX"、"现在有几张表"。

## 操作顺序

1. 先调 `current_document` 取得当前 doc_id，绝不猜测路径。
2. 用 `view_stats` 获得整体规模（段落数、表格数）。
3. 按需用 `view_body`/`view_table`/`search_*` 定位具体位置，给出段落/表格索引。
4. 汇报事实与引用（第 N 段、第 M 行第 K 列），不臆测未读到的内容。

## 边界

- 只做只读调查，不调用任何写工具，不触发保存，不改变文档 dirty 状态。
- 不伪造未读到的段落内容；读不到就如实说"未找到"。
- 引用位置必须来自工具返回的真实索引，不编造、不四舍五入行列号。
- 大文档不必一次读完所有段落；按用户关注点分批定位即可，但要说明读取范围。

## 停止 / 报告条件

- 用户得到结构化的事实与位置引用后，本轮调查即完成。
- 若用户意图其实是修改，等待其明确改动请求后再进入编辑流程；本 Skill 不主动改写。
- 当工具返回结构与你预期不符（如表格嵌套、空段）时，以工具事实为准并简要说明，不强行解读。
