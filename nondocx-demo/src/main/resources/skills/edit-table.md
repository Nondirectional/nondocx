# edit-table

## 触发场景

用户要求处理表格：创建表格、改单元格、合并单元格、调整表格布局。例如：
"插入一个 3x3 的表"、"把第一行第一列改成 X"、"合并这两个单元格"。

## 操作顺序

1. 先 `current_document` 取 doc_id；涉及现有表格时先 `view_table` 确认表格索引与行列结构。
2. 按意图调写工具：`create_table`/`set_cell_text`/`merge_cells` 等表格相关工具。
3. 改完用 `review_intent` 复审用户期望的表格改动是否已达成。
4. 保存由应用层在循环结束时强制执行，你不持有保存工具。

## 边界

- 只负责表格结构与单元格；页眉页脚写入不属于本 Skill，本 Demo 暂不支持。
- 写工具调用标记 dirty，触发应用层落盘（复审为软警告，不拦截保存）。
- 表格结构偏差可在复审后叠加 `audit-quality` 解释与补改；模型可自行选择组合与否。
- 改动单元格前先确认目标表格索引与行列号，行列从 0 起按工具返回为准。

## 停止 / 报告条件

- 表格改动完成并复审后结束；复审为软警告，文档总会保存，据实转述结论即可。
- 合并/拆分遇到 POI 限制无法完成时，如实说明限制，不伪造结果。
- 新建表格应给出明确行列数；用户未指定行列时先询问，不默认建空表。
